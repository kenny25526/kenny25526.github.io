EDU SRC 挖洞实战：我常用的打法

发表于2026-03-03|更新于2026-03-04|红队攻防web安全

最近一直被催着分享之前挖 CNVD 的思路，今天整理一篇我平时挖 EDU 的实战路线。
文中截图都是真实案例，目标均已修复，仅用于技术复盘。

1. 先说核心：优先盯小程序

很多人还在主打 Top10、绕 WAF、社工账号密码，这些当然也能出洞。
但我自己更常用的路线是：先从小程序入口拿到身份凭据，再去跑业务接口找敏感数据。

2. 常见框架的切入思路

先看数据包和路径特征，常见的几个框架基本都能快速识别：若依、Jeecg、Admin.NET、Spring Boot。

2.1 若依（Ruoyi）

常见特征路径：/prod-api、/xxx-api。

  1. 常见弱口令：ruoyi/admin123、ry/admin123

一个弱口令，轻则低分，重则高危。
进后台后可以继续看计时任务、Nday 点位、前台 Shiro 等可利用面。

  1. 小程序一键登录拿到 JWT 后，直接跑敏感接口

跑到身份证号、手机号等批量敏感信息，通常就能上高危。
若依里也经常能遇到“获取初始密码”这类危险接口。

  1. 没有 Session 时，也可以顺手看 Druid 弱口令

2.2 Jeecg

常见路径：/jeecg-boot/、/xxxx-boot/。

常见思路：

  1. 登录拿 JWT 后跑敏感接口
  2. 尝试弱口令（例如 jeecg/123456）
  3. 后台排查积木报表等 SQL 注入点

此外，部分系统会有普通用户提权到管理员的接口，值得重点梳理。

2.3 Admin.NET

常见接口前缀：/api/sysxxxxxx。
套路类似，拿到凭据后批量梭哈接口，经常能摸到敏感证书或配置文件。

2.4 Spring Boot

  1. 先看经典报错页

  1. 再看 Swagger 暴露情况

接口一个个测，未授权总会遇到。
注意控制测试范围，避免误触删除类接口。

3. 我在小程序里高频拿分的点

我平时不少分数都来自小程序业务逻辑，尤其是“任意重置密码后继续横向点功能”这条线。

3.1 登录逻辑漏洞

  1. 重点看登录请求包是否泄露 sessionKey，用于解密数据或伪造登录态

  1. 越权 + 登录逻辑组合，直接做到全站用户接管

越权返回里如果有 openid、unionid、手机号等字段，就继续联动登录接口。

当登录只校验“手机号 + openid + unionid”三要素时，拿到这三项就能登录他人账号。

3.2 登录后的头像上传

这个点经常被低估，文件上传策略一旦松，后续利用空间很大。

实战里建议先做可控回显验证，按授权范围推进。

3.3 验证码绕过

  1. 验证码和手机号都在请求包里可改

把手机号改成自己的，再填自己收到的验证码，经常可直接绕过。

3.4 手机号爆破

这类接口如果没有频控和风控策略，风险很容易放大。

4. 总结

这篇主要是复盘我平时挖 EDU 的一条实战思路：
小程序入口 -> 身份凭据/JWT -> 业务接口联动 -> 敏感数据与权限问题。

每个功能点多点几下，多做接口之间的关联验证，通常比单点盲测更容易出结果。
