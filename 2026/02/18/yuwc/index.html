PTT (Pass The Ticket - 票据传递攻击)

PTT (Pass The Ticket) 是基于 Kerberos 协议的票据传递攻击技术，通过窃取或伪造 Kerberos 票据来实现权限维持和横向移动。

Kerberos 协议简介

Kerberos 是一种网络认证协议，其设计目标是通过密钥系统为客户端/服务器应用程序提供强大的认证服务。该认证过程具有以下特点：

  • 不依赖于主机操作系统的认证
  • 无需基于主机地址的信任
  • 不要求网络上所有主机的物理安全
  • 假定网络上传送的数据包可以被任意读取、修改和插入

PTT 攻击方式

PTT 中最常见的三种攻击方式：

  1. 黄金票据 (Golden Ticket) - 伪造 TGT 票据
  2. 白银票据 (Silver Ticket) - 伪造 TGS 票据
  3. MS14-068 - 利用 KDC 漏洞提权

黄金票据 (Golden Ticket)

黄金票据是通过伪造 krbtgt 用户的 TGT 票据来实现域内权限维持的技术。krbtgt 是域控中用来管理和发放票据的特殊用户，拥有该用户的权限就可以伪造域内任意用户的票据。

黄金票据的优势

即使域控主机权限丢失或管理员密码被修改，依然可以使用边缘主机上的黄金票据模拟获得域内最高权限，实现持久化控制。

特点：

  • 票据有效期长（默认 10 年）
  • 不依赖域控在线状态
  • 可访问域内所有资源
  • 难以被检测和清除

白银票据 (Silver Ticket)

白银票据是通过伪造服务账户的 TGS (Ticket Granting Service) 票据来访问特定服务的攻击技术。与黄金票据不同，白银票据只能访问特定的服务，但更加隐蔽。

白银票据 vs 黄金票据

特性 黄金票据 白银票据
伪造对象 TGT (krbtgt) TGS (服务账户)
访问范围 域内所有资源 特定服务
所需权限 域控权限 服务账户哈希
隐蔽性 较低 较高
检测难度 容易 困难

适用场景：

  • 针对特定服务的持久化访问
  • 避免触发域控的检测机制
  • 在无法获取 krbtgt 哈希时的替代方案

MS14-068 漏洞利用

MS14-068 是密钥分发中心 (KDC) 服务中的 Windows 漏洞，该漏洞允许经过身份验证的用户在其 Kerberos 票据 (TGT) 中插入任意 PAC (Privilege Attribute Certificate)。

漏洞原理

该漏洞位于域控制器的 kdcsvc.dll 中，攻击者可以通过呈现具有篡改 PAC 的 Kerberos TGT 来获得域管理员权限的票据。

利用条件

  • 域内普通用户权限
  • 目标域控未打补丁 (KB3011780)
  • 知道用户的明文密码
  • 获取用户的 SID

票据传递的优势

  • 无需明文密码或哈希值
  • 绕过多因素认证 (MFA)
  • 可在多台主机间复用
  • 隐蔽性强，难以检测

防御建议

针对黄金票据：

  1. 定期更换 krbtgt 账户密码（需更换两次）
  2. 监控异常的 TGT 请求
  3. 启用 PAC 验证

针对白银票据：

  1. 定期更换服务账户密码
  2. 监控异常的服务票据请求
  3. 限制服务账户权限

针对 MS14-068：

  1. 及时安装安全补丁 KB3011780
  2. 升级到 Windows Server 2012 R2 及以上版本

通用防御措施：

  1. 部署 EDR 监控 Kerberos 异常行为
  2. 启用 Windows Defender Credential Guard
  3. 限制域管理员账户的使用范围
  4. 定期审计域内高权限账户
  5. 监控 Event ID 4768、4769、4770 等 Kerberos 相关日志
