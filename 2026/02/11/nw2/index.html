内网横向移动

发表于2026-02-11|更新于2026-02-12|红队攻防
|浏览量:

含义

在内网渗透中，当攻击者获取到内网某台机器的控制权后，会以被攻陷的主机为跳板，通过收集域内凭证等各种方法，访问域内其他机器，进一步扩大资产范围。通过横向，攻击者最终获得域控制器的访问权限，甚至完全控制基于 Windows 操作系统的整个内网环境，控制域环境下的全部机器。
一般分为四个点：1.漏洞利用 2.服务利用 3.令牌横向 4.软件部署利用

漏洞利用

ms17_010 最好不要打蓝屏，直接用 msf 一把梭 auxiliary/admin/smb/ms17_010_command

内网典型漏洞利用清单：操作系统服务漏洞：ms08_067、ms17_010、cve2019_0708。操作系统弱口令：SSH、RDP、VNC、telnet。各类数据库漏洞：mysql、mssql、redis。各类中间件框架漏洞：struts2 系列、weblogic 系列、Jboss 系列、Tomcat 系列、Jekins 系列、Apache Solr。各类开源框架漏洞：fastjson、shiro。各类开源系统漏洞：禅道项目管理系统、Jumpserver。各类商业系统漏洞：泛 OA、金 OA、蓝 OA、致 OA。

ipc横向

IPC(Internet Process Connection)共享命名管道的资源，是为了实现进程间通信而开放的命名管道。IPC 可以通过验证用户名和密码获得相应的权限，使用 139/445 端口。利用条件：1、目标机开启了 139/445 端口 2、目标主机管理员开启了 ipc$ 默认共享 3、知道目标机的账户密码

WMI横向

WMI 全称“Windows 管理规范”，从 win2003 开始一直存在。它原本的作用是方便管理员对 Windows 主机进行管理，因此在内网渗透中，我们可以使用 WMI 服务进行横向移动。其支持用户名明文或 hash 的方式进行认证，并且该方法不会在目标日志系统留下痕迹。
利用条件：1、WMI 服务开启，端口 135，默认开启。2、防火墙允许 135/445 等端口通信。3、知道目标机的账户密码。
自带 WMI 明文传递，无回显。

Impacket 工具：wmiexec.py

smb横向

SMB 是同一网络或域上的 Windows 机器的文件、打印机和串行端口共享协议。使用 SMB 与文件共享进行交互，可以在整个网络中横向移动。利用 SMB 服务可以通过明文或 hash 传递来远程执行，条件是 445 服务端口开放。利用条件：1、445 端口开启 2、知道账号密码

DCOM横向

Windows 组件对象模型（COM）是原生 Windows 应用程序编程接口（API）的一个组件，它允许软件对象或实现一个或多个接口的可执行代码之间进行交互。通过 COM，客户端对象可以调用服务器对象的方法，通常是动态链接库（DLL）或可执行文件（EXE）。分布式 COM（DCOM）是透明的中间件，使用远程过程调用（RPC）技术将 COM 的功能扩展到本地计算机之外。通过利用分布式组件对象模型（DCOM）使用有效帐户与远程机器进行交互。默认情况下，只有管理员才能通过 DCOM 远程激活和启动 COM 对象。

WinRM横向

WinRM（微软远程管理）是 WS-Management 协议的实现组件。WinRM 是 Windows 操作系统的一部分，是一项允许管理员在系统上远程执行管理任务的服务，使用此服务需要管理员级别的凭据。通信通过 HTTP（5985）或 HTTPS（5986）执行，5986 关闭表示 WinRM 服务配置为仅接受 HTTP 连接。默认情况下支持 Kerberos 和 NTLM 身份验证以及基本身份验证。Windows Vista/Win7 必须手动启动，Windows 2008 以上版本默认自动状态；WinRS 适用于 Windows 2008 以上版本，Windows 2012 之后的版本默认允许远程任意主机来管理。

PSEXEC横向

PsExec 主要用于大批 Windows 主机的维护，在域环境下效果尤其好（因为 PsExec 是 Windows 提供的工具，所以杀毒软件将其列入白名单中）。因为使用 PsExec 通过命令行环境与目标机器建立连接，甚至控制目标机器，而不需要通过远程桌面（RDP）进行图形化的控制。PsExec 包含在 PsTools 工具包中，下载地址：https://download.sysinternals.com/files/PSTools.zip。通过 PsExec 可以在远程目标主机上执行命令，也可以将管理员权限提升为 System 权限以运行指定的程序。PsExec 的基本原理是：通过管道在远程目标主机上创建一个 psexec 服务，并在本地磁盘中生成一个名为 “PSEXESVC” 的二进制文件，然后通过 psexec 服务运行命令，运行结束后删除服务。
