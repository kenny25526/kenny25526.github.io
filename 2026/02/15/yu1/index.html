PTH 之 AT 计划任务横向

AT 命令用于在远程主机上创建计划任务执行命令，适用于 Windows Server 2008 及更早版本。

优点： 无需交互式 Shell，适合执行单条命令
缺点： 新版 Windows 已弃用 AT 命令，推荐使用 schtasks

PTH 之 WMI 横向

WMI (Windows Management Instrumentation) 是 Windows 系统管理的核心组件，可远程执行命令且无文件落地。

特点：

  • 无需写入磁盘，隐蔽性强
  • 需要目标开放 135 端口（RPC）
  • 需要管理员权限

PTH 之 SMB 横向

通过 SMB 协议远程执行命令，原理是通过 SMB 上传可执行文件到目标系统并执行。

特点：

  • 需要目标开放 445 端口
  • 会在目标系统创建服务（可能触发告警）
  • 执行完毕后自动清理服务

PTH 之 DCOM 横向

DCOM (Distributed Component Object Model) 是 Windows 分布式组件对象模型，可用于远程代码执行。

常用 DCOM 对象：

  • MMC20.Application
  • ShellWindows
  • ShellBrowserWindow

优点： 相比 WMI 更隐蔽，日志记录较少

PTH 之 PSEXEC 横向

PSExec 是 Sysinternals 套件中的经典工具，通过 SMB 协议在远程系统创建服务执行命令。

工作流程：

  1. 通过 SMB 连接到目标的 ADMIN$ 共享
  2. 上传 PSEXESVC.exe 到 System32 目录
  3. 通过 SCM (Service Control Manager) 创建并启动服务
  4. 执行完毕后删除服务

注意事项：

  • 会产生大量日志（Event ID 7045）
  • 需要目标开放 445 端口
  • 需要管理员权限

PTK (Pass The Key - 密钥传递攻击)

攻击背景

当目标系统满足以下条件时，传统的 PTH 攻击将失效：

  • 安装了 KB2871997 补丁
  • 禁用了 NTLM 认证协议
  • 强制使用 Kerberos 认证

此时可以使用 Pass The Key 攻击，通过 AES 密钥进行 Kerberos 认证。

PTH vs PTK 的区别

攻击方式 使用凭证 加密方式 认证协议
Pass The Hash NTLM Hash RC4 NTLM
Pass The Key AES Key (128/256) AES Kerberos

  • AES 加密强度更高，不易被检测

适用场景：

  • 高安全环境（禁用 NTLM）
  • 域控强制 Kerberos 认证
  • 需要更隐蔽的横向移动

PTT (Pass The Ticket - 票据传递攻击)

票据传递攻击是 Kerberos 认证体系中的高级攻击技术，通过窃取或伪造 Kerberos 票据实现权限维持和横向移动。

内容预告：

  • Golden Ticket（黄金票据）
  • Silver Ticket（白银票据）
  • TGT/TGS 票据注入
  • 票据导出与重放
  • Kerberos 委派攻击

详细内容将在下一篇文章《域渗透之权限维持》中深入讲解。

总结

本文介绍了域环境中常见的横向移动技术：

Pass The Hash (PTH)：

  • IPC、AT、WMI、SMB、DCOM、PSEXEC、RDP 等多种方式
  • 适用于 NTLM 认证环境
  • 需要管理员权限

Pass The Key (PTK)：

  • 使用 AES 密钥进行 Kerberos 认证
  • 绕过 NTLM 禁用限制
  • 更隐蔽，适合高安全环境

Pass The Ticket (PTT)：

  • 基于 Kerberos 票据的攻击
  • 下篇文章详细展开

防御建议：

  1. 启用 LSASS 保护（PPL）
  2. 禁用 NTLM 认证，强制 Kerberos
  3. 限制管理员账户的横向移动
  4. 部署 EDR 监控异常登录行为
  5. 定期轮换高权限账户密码
  6. 启用 Windows Defender Credential Guard
