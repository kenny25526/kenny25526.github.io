Exchange特殊组

安装Exchange后默认会⾃动添加⼀个名为Microsoft Exchange Security Groups的OU
包括两个特殊的组：
Exchange Trusted Subsystem 和 Exchange Windows Permission


默认情况下， Exchange Windows Permissions 对安装Exchange的域对象具有WriteDACL权限， Exchange
Trusted Subsystem 继承WriteDACL权限。
“WriteDACL权限” 代表对对象的访问控制列表（DACL）进⾏写⼊操作的权限。
DACL 是⽤于控制对象（如⽂件、⽂件夹、注册表项等）的访问权限的列表。
具有 WriteDACL 权限的⽤户或组可以修改对象的访问控制列表，从⽽控制其他⽤户对该对象的访问权限。
通过 WriteDACL 权限，⽤户可以执⾏以下操作：

  1. 添加或删除访问控制条⽬（ACE）：⽤户可以添加或删除对象的访问控制条⽬，以授予或撤销其他⽤户对该对
    象的访问权限。
  2. 修改访问控制条⽬：⽤户可以修改对象的访问控制条⽬，包括更改权限、编辑⽤户或组的成员资格等。
  3. 修改访问控制规则：⽤户可以修改对象的访问控制规则，如继承规则、继承层次结构等。

  4. 控制对象的安全性：⽤户可以通过修改对象的访问控制列表来控制对象的安全性，确保只有授权的⽤户能够
    访问该对象。
