内网横向移动技术

发表于2026-02-07|更新于2026-02-13|红队攻防

横向移动概述

在内网渗透中，当攻击者获取到内网某台机器的控制权后，会以被攻陷的主机为跳板，通过收集域内凭证、利用服务漏洞、滥用系统特性等方法，访问域内其他机器，进一步扩大资产范围。

横向移动的最终目标通常是获得域控制器的访问权限，从而完全控制基于 Windows 的整个内网环境。

横向移动的四大维度

  1. 漏洞利用：利用操作系统或服务的已知漏洞（如 MS17-010、CVE-2019-0708）
  2. 服务利用：滥用合法的 Windows 服务（如 SMB、WMI、WinRM、PSEXEC）
  3. 令牌横向：窃取和滥用访问令牌实现权限提升和横向
  4. 软件部署利用：利用企业部署工具（如 SCCM、GPO）进行批量横向

漏洞利用横向

MS17-010（永恒之蓝）

漏洞概述

MS17-010 是 SMB 协议的远程代码执行漏洞，影响 Windows XP 至 Windows Server 2016 的多个版本。该漏洞无需认证即可触发，危害极大。

利用注意事项

警告：MS17-010 利用可能导致目标系统蓝屏崩溃，在生产环境中需谨慎使用。

漏洞编号 影响系统 利用难度 危害等级

操作系统弱口令

  • SSH：Linux 系统的远程管理协议（22 端口）
  • RDP：Windows 远程桌面（3389 端口）
  • VNC：跨平台远程桌面（5900 端口）
  • Telnet：明文传输的远程管理协议（23 端口）

数据库漏洞与弱口令

  • MySQL：默认 3306 端口，常见弱口令 root/root
  • MSSQL：默认 1433 端口，支持 xp_cmdshell 命令执行
  • Redis：默认 6379 端口，未授权访问可写 SSH 密钥
  • MongoDB：默认 27017 端口，未授权访问可导出数据

中间件与框架漏洞

  • Struts2 系列：S2-001/S2-045/S2-057 等 RCE 漏洞
  • Weblogic 系列：反序列化、SSRF、任意文件上传
  • JBoss 系列：JMX Console 未授权访问
  • Tomcat 系列：弱口令、war 包部署
  • Jenkins：未授权访问、脚本命令执行

开源框架与系统漏洞

  • Fastjson：反序列化 RCE（多个版本）
  • Shiro：反序列化 RCE（RememberMe）
  • 禅道项目管理系统：SQL 注入、文件上传
  • Jumpserver：未授权访问、RCE
  • 泛微 OA / 金蝶 OA / 蓝凌 OA / 致远 OA：SQL 注入、文件上传、命令执行

IPC 横向移动

原理

IPC（Internet Process Connection）是 Windows 为进程间通信开放的命名管道资源。通过 IPC$ 共享，可以使用管理员凭证远程执行命令。

利用条件

  1. 目标机开启 139/445 端口
  2. 目标主机管理员开启 IPC$ 默认共享
  3. 已知目标机的管理员账户密码

WMI 横向移动

原理

WMI（Windows Management Instrumentation）是 Windows 的管理框架，从 Windows 2003 开始内置。WMI 支持远程管理，可以使用明文密码或 NTLM Hash 进行认证。

利用条件

  1. WMI 服务开启（端口 135，默认开启）
  2. 防火墙允许 135/445 端口通信
  3. 已知目标机的管理员凭证

WMI 横向的优势

  • 支持 Pass-the-Hash 攻击
  • 不会在目标系统留下明显的服务或进程
  • 可以批量执行命令
  • 支持远程 PowerShell 脚本执行

检测与防御

蓝队检测：

  • 监控 WMI 进程创建事件（Event ID 4688）
  • 检测异常的 WmiPrvSE.exe 进程
  • 分析网络流量中的 DCOM 连接

红队规避：

  • 使用 PowerShell 的 WMI Cmdlet（避免 wmic.exe 特征）
  • 限制执行频率，避免批量触发告警
  • 结合其他横向方法分散特征

  2. 已知管理员账号密码或 NTLM Hash

  • 需要管理员权限

DCOM 横向移动

原理

DCOM（Distributed Component Object Model）是 COM 的分布式扩展，允许通过网络调用远程对象的方法。通过 DCOM 可以在远程机器上执行命令。

利用条件

  1. 目标机开启 DCOM（默认开启）
  2. 防火墙允许 135 端口
  3. 已知管理员凭证

WinRM 横向移动

原理

WinRM（Windows Remote Management）是 Windows 的远程管理协议，基于 WS-Management 标准。WinRM 允许管理员远程执行命令和脚本。

利用条件

  1. WinRM 服务开启（HTTP 5985 / HTTPS 5986）
  2. Windows 2008 以上版本
  3. 已知管理员凭证

PSEXEC 横向移动

原理

PsExec 是 Sysinternals 套件中的工具，用于远程执行命令。其原理是通过 SMB 协议在远程机器上创建服务，然后执行指定程序。

利用条件

  1. 445 端口开启
  2. 已知管理员凭证
  3. 目标机允许文件共享

PsExec 的工作流程

  1. 通过 SMB 连接到目标机的 ADMIN$ 共享
  2. 上传 PSEXESVC.exe 到目标机的 System32 目录
  3. 通过 SCM（服务控制管理器）创建并启动服务
  4. 服务执行指定的命令或程序
  5. 执行完毕后删除服务和文件

检测与防御

蓝队检测：

  • 监控 PSEXESVC 服务的创建（Event ID 7045）
  • 检测 ADMIN$ 共享的异常访问
  • 分析 System32 目录的文件创建事件

红队规避：

  • 使用自定义服务名称
  • 修改 PSEXESVC.exe 的文件名
  • 使用其他横向方法替代

横向移动最佳实践

红队建议

  1. 凭证收集优先：

    • 使用 Mimikatz / LaZagne 收集凭证
    • 导出 NTDS.dit 获取所有域用户 Hash
    • 收集浏览器保存的密码

  2. 选择合适的横向方法：

    • 隐蔽性要求高：WMI、DCOM
    • 需要交互式 Shell：WinRM、RDP
    • 批量横向：CrackMapExec、Spray

  3. 避免触发告警：

    • 限制横向频率，避免短时间大量登录
    • 使用合法的管理工具（WinRM、PowerShell Remoting）
    • 避免使用已知的恶意工具特征

  4. 建立多条通道：

    • 部署多个 Beacon（HTTP、SMB、DNS）
    • 在关键节点部署持久化
    • 准备备用的横向路径

蓝队防御

  1. 网络隔离：

    • 限制工作站之间的横向通信
    • 使用防火墙策略限制 SMB、WMI、RDP
    • 部署网络访问控制（NAC）

  2. 凭证保护：

    • 启用 Credential Guard
    • 禁用 NTLM 认证，强制使用 Kerberos
    • 定期更换管理员密码
    • 使用 LAPS 管理本地管理员密码

  3. 日志监控：

    • 监控 Event ID 4624（登录成功）
    • 监控 Event ID 4672（特权登录）
    • 监控 Event ID 7045（服务创建）
    • 监控 Event ID 4688（进程创建）

  4. 端点防护：

    • 部署 EDR 解决方案
    • 启用 Windows Defender ATP
    • 限制 PowerShell 执行策略
    • 启用 AppLocker / WDAC

总结

内网横向移动是渗透测试的核心环节，成功的横向依赖于：

  1. 充分的信息收集：了解网络拓扑、主机角色、凭证信息
  2. 合适的工具选择：根据目标环境选择最隐蔽的横向方法
  3. 谨慎的操作策略：避免触发告警，保持长期访问能力
  4. 完善的应急预案：准备多条横向路径和备用通道

记住：横向移动技术需要在授权的渗透测试环境中使用，未经授权的横向行为属于违法行为。
