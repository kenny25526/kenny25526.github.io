Windows 提权技术

在 Windows 系统中获取初始访问权限后，通常需要将权限提升至 SYSTEM 或管理员级别，以便执行更深入的渗透操作。以下是常见的 Windows 提权技术。

土豆家族提权

原理概述

“土豆家族”是一系列利用 Windows 本地权限提升漏洞的技术集合，核心原理是滥用 Windows 的 NTLM 认证机制和 COM 对象，诱骗 SYSTEM 权限的进程向攻击者控制的服务进行身份验证，从而窃取 SYSTEM 令牌。

土豆家族演进

技术名称 适用系统 核心原理 利用条件
Hot Potato Windows 7/8/10/Server 2008/2012 NBNS 欺骗 + NTLM 中继 SeImpersonate 或 SeAssignPrimaryToken 权限
Rotten Potato Windows 7/8/10/Server 2008/2012/2016 COM 对象劫持 + NTLM 中继 SeImpersonate 或 SeAssignPrimaryToken 权限
Juicy Potato Windows 7/8/10/Server 2008/2012/2016 COM 对象劫持（扩展版） SeImpersonate 或 SeAssignPrimaryToken 权限
Rogue Potato Windows 10/Server 2016/2019 伪造 OXID 解析器 SeImpersonate 或 SeAssignPrimaryToken 权限
Sweet Potato Windows 10/Server 2016/2019/2022 综合多种技术 SeImpersonate 或 SeAssignPrimaryToken 权限
God Potato Windows 10/Server 2016/2019/2022 最新综合利用 SeImpersonate 或 SeAssignPrimaryToken 权限

MySQL 提权

MySQL 的 UDF（User Defined Function，用户自定义函数）功能允许用户通过动态链接库扩展 MySQL 功能。攻击者可以上传恶意 UDF 库，创建自定义函数来执行系统命令。

UDF 提权原理

  1. 将恶意 UDF 动态链接库（.dll 或 .so）上传到 MySQL 的 plugin 目录
  2. 在 MySQL 中创建自定义函数，指向该库
  3. 调用自定义函数执行系统命令

SUID 提权

原理

SUID（Set User ID）是 Linux 的一种特殊权限，允许用户以文件所有者的权限执行该文件。如果一个 root 拥有的程序设置了 SUID 位，普通用户执行时会以 root 权限运行。

总结

提权是内网渗透的关键环节，成功的提权依赖于：

  1. 信息收集：全面收集系统信息、权限配置、运行服务
  2. 漏洞识别：使用自动化工具快速定位可利用的提权路径
  3. 灵活利用：根据目标环境选择合适的提权技术
  4. 权限维持：提权后及时建立持久化机制
  5. 痕迹清理：清理日志和命令历史，降低被发现风险

记住：提权技术需要在授权的渗透测试环境中使用，未经授权的提权行为属于违法行为。
