DLL 劫持原理

什么是 DLL 劫持

DLL 劫持（DLL Hijacking）是一种利用 Windows DLL 搜索顺序机制的攻击技术。当应用程序调用 LoadLibrary() 或 LoadLibraryEx() 加载 DLL 时，如果未指定完整路径，Windows 会按照预定义的搜索顺序查找 DLL 文件。攻击者可以在优先级更高的目录中放置恶意 DLL，使系统加载恶意代码而非合法 DLL。

白加黑技术

白加黑（Living off the Land） 是指利用合法的、已签名的可执行文件（白文件）加载恶意 DLL（黑文件），从而绕过安全软件的检测。这种技术的优势在于：

  1. 信任链利用：合法程序通常具有数字签名，被安全软件信任
  2. 行为隐蔽：恶意行为由合法进程执行，难以被行为检测识别
  3. 免杀效果好：不需要修改白文件，只需替换 DLL
  4. 持久化简单：可以通过快捷方式、计划任务等方式持久化

Windows DLL 搜索顺序

标准搜索顺序

当应用程序调用 LoadLibrary("example.dll") 时，Windows 按以下顺序搜索 DLL：

  1. 应用程序所在目录（最高优先级）

    • 例如：C:\Program Files\App\example.dll

  2. 系统目录（GetSystemDirectory()）

    • 32 位系统：C:\Windows\System32
    • 64 位系统：C:\Windows\System32（64 位 DLL）
    • 64 位系统：C:\Windows\SysWOW64（32 位 DLL）

  3. 16 位系统目录

    • C:\Windows\System（已废弃，兼容性保留）

  4. Windows 目录（GetWindowsDirectory()）

    • C:\Windows

  5. 当前工作目录（GetCurrentDirectory()）

    • 进程启动时的目录

  6. PATH 环境变量中的目录

    • 按顺序依次搜索

安全 DLL 搜索模式

Windows 提供了 SafeDllSearchMode（默认启用），调整搜索顺序为：

  1. 应用程序所在目录
  2. 系统目录
  3. 16 位系统目录
  4. Windows 目录
  5. 当前工作目录（优先级降低）
  6. PATH 环境变量目录

注册表位置：

  • 1：启用（默认）
  • 0：禁用

3. 调试技巧

如果程序崩溃：

  1. 检查架构匹配：确保 DLL 和 EXE 的位数一致（x86/x64）
  2. 检查导出函数：使用 dumpbin /exports version.dll 验证导出表
  3. 检查原始 DLL 路径：确保 version_original.dll 存在
  4. 使用调试器：x64dbg 或 WinDbg 定位崩溃点

如果未执行 Shellcode：

  1. 检查 DLL 是否被加载：使用 Process Explorer 查看
  2. 检查 Shellcode 格式：确保是 C 数组格式（\xfc\x48...）
  3. 检查内存权限：确保使用 PAGE_EXECUTE_READWRITE

防御检测与对抗

蓝队检测手段

  1. DLL 签名验证：检查 DLL 是否有合法数字签名
  2. DLL 路径检测：监控非系统目录加载的 DLL
  3. 行为分析：检测 DLL 加载后的异常网络连接
  4. Sysmon 监控：记录 ImageLoad 事件（Event ID 7）
  5. EDR 检测：检测 VirtualAlloc + memcpy + 线程创建模式

红队对抗策略

  1. 使用合法签名：购买代码签名证书或窃取证书
  2. 内存加载：不在磁盘留下原始 DLL
  3. 延迟执行：避免在 DLL 加载时立即执行
  4. 加密 Shellcode：运行时解密，避免静态检测
  5. 混淆导出表：使用序号导出而非函数名
  6. 选择低关注度白文件：避免使用知名软件

实战案例

案例一：微信 DLL 劫持

目标：劫持微信的 version.dll

步骤：

  1. 使用 Procmon 确认微信加载 version.dll
  2. 生成代理 DLL，转发所有导出函数
  3. 在 DllMain 中加载 Cobalt Strike Beacon
  4. 将恶意 DLL 和原始 DLL 放入微信目录
  5. 启动微信，成功上线

优势：

  • 微信是高频使用软件，持久化效果好
  • 微信有数字签名，不易被拦截

案例二：WPS Office DLL 劫持

目标：劫持 WPS 的 dwmapi.dll

步骤：

  1. 分析 WPS 启动时加载的 DLL

  4. 通过钓鱼邮件发送包含恶意 DLL 的 WPS 安装包
  5. 用户安装后，每次启动 WPS 都会触发

总结

DLL 劫持（白加黑）是一种高效的免杀技术，通过利用 Windows DLL 搜索机制和合法程序的信任链，可以绕过大部分安全软件的检测。实战中需要注意：

  1. 选择合适的白文件：有签名、高频使用、低关注度
  2. 完整转发导出函数：避免程序崩溃
  3. 隐蔽执行 Shellcode：延迟执行、加密、反沙箱
  4. 清理痕迹：避免在磁盘留下多余文件
  5. 持续更新：对抗不断升级的检测规则

记住：DLL 劫持的成功率取决于对目标环境的深入分析和对 Windows 加载机制的理解。
